# Architecture

## Data pipeline (bottom-up)

### Ingest
- `scripts/pull_noaa.py` → `data/raw/noaa_kphx_2022-06_2023-10.csv` (temp_c, dewpoint_c, wind_mps, slp_hpa)
- `scripts/pull_eia930.py` → `data/raw/eia930_azps_2022-06_2023-10.csv` (hourly demand MW)
- `scripts/pull_ieee123.sh` → `data/ieee123/` (OpenDSS master + BusCoords)
- `scripts/nws_fetch.py::fetch_phoenix_hourly()` → live NWS tomorrow (no auth, User-Agent required)

### Featurize
- `src/gridsense/features.py::build_hourly_features(start, end, data_root)`
  - Returns `FeatureBundle(times, node_names, X_exog[T,11], X_node[T,132,1], y_kw[T,132], scalers, meta)`
  - `X_exog` = `temp_c, dewpoint_c, wind_mps, slp_hpa, hour_sin, hour_cos, dow_sin, dow_cos, is_weekend, month_sin, month_cos`
  - `X_node` = last-hour-load z-scored via the `y_kw` scalers
  - `y_kw` = RAW kW (no scaling — contract with predictor, see commit `c31f18d`)

### Train
- `scripts/train.py` — fits GWNet for 200 epochs on A100 (Colab)
- Outputs:
  - `data/models/gwnet_v0.pt` (checkpoint, 263 KB, 59,890 params, **tracked in git**)
  - `data/models/metrics.json` (final train/val/test MAE)
  - `data/models/history.json` (per-epoch history)

### Predict
- `src/gridsense/predictor.py`
  - `load_predictor(ckpt_path, metrics_path, device) -> LoadedPredictor`
  - `forecast_from_bundle(predictor, bundle, t_in=24, t_out=6) -> Forecast(p10, p50, p90, timestamps, bus_names)`
  - Output is RAW kW (no denormalisation bug — fixed in `c31f18d`)
  - Rolls 4× to cover 24 h from 6 h horizon

### Scenario transforms
- `src/gridsense/decision.py`:
  - `heat_wave_scenario(demand_multiplier=1.4)` — shifts exogenous `temp_c` +5.56 °C and applies a cooling-load profile on hours 12-22 local
  - `ev_surge_scenario(ev_fleet_size=2000)` — adds +720 kW at 20 deterministic residential buses during hours 17-22 local
  - `rank_critical_buses(voltages, loadings, topk=10)` — voltage deviation + loading fraction heuristic
  - `recommend_actions(scenario) -> list[str]` — deterministic operator playbook

### Power flow
- `src/gridsense/power_flow.py::run_snapshot(root, overrides) -> SnapshotResult(converged, bus_voltages_pu, line_loadings_pct, total_losses_kw)`
- Wraps OpenDSSDirect.py against IEEE 123 master. Snapshot (not timeseries) — we sample the scenario's peak hour.

### Precompute (orchestrator)
- `scripts/precompute_forecasts.py` stitches all the above and writes 5 JSONs per run into `web/public/data/forecasts/`:
  - `tomorrow_baseline.json` — raw NWS-driven forecast
  - `tomorrow_heat.json` — +10 °F + cooling profile
  - `tomorrow_ev.json` — +720 kW × 20 buses
  - `feeder_topology.json` — bus coords + line connectivity for the map
  - `model_metrics.json` — train/val/test MAE for the ModelMetrics component
  - (`generated_at.json` timestamp-only side file)

### State flow
```
ScenarioProvider mount → Promise.all 6 fetches → validate → ready
  → children see useScenario() = {active, compareWith, baseline, heat, ev, topology, metrics, current}
  → B/H/E keys set active, C key cycles compareWith through non-active scenarios
  → components re-render instantly (no refetch, all preloaded)
```

## Contracts worth memorising

- `y_kw` is always **raw kW**, never scaled. Scalers in the bundle apply only to `X_node` (last-hour-load history).
- Predictor outputs quantiles in the same raw-kW space. Any `* scaler.std + scaler.mean` you see in predictor code is a bug (the c31f18d fix removed one).
- `peak_hour` on every forecast JSON is **local Phoenix hour** (America/Phoenix, no DST). If you see a UTC hour leak, that's a regression of the d378b80 fix.
- Heat scenario must produce `peak_mw >= 1.3 * baseline.peak_mw` — asserted by `web/lib/validate.ts`. EV scenario must produce peak hour ∈ `[17, 22]` local — also asserted.
- Precompute JSON schema is owned by `web/lib/types.ts`. Change both sides together or validation will red-line the UI.
